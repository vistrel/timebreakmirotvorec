# MineCatalog MVP

Простой каталог персоналий на Django с готовой админкой и пиксельным интерфейсом в духе Minecraft.

## Почему этот стек
- **Django** — быстрый старт, встроенная админка, аутентификация, формы, CSRF-защита.
- **Django Admin** — не нужно писать отдельную админ-панель с нуля.
- **SQLite** — идеален для MVP и локального запуска.
- **Обычные Django-шаблоны + CSS** — минимум сложности, один проект без отдельного frontend/backend.

## Что уже есть
- Главная страница со списком карточек
- Поиск
- Фильтрация по категориям
- Пагинация
- Страница категории
- Страница карточки
- Статические страницы
- SEO title / description
- Slug URL
- Статус публикации
- Флаг показа на главной
- Настройки сайта
- Текстовые блоки для редактирования через админку
- Загрузка изображений
- Базовые тесты

## Быстрый запуск

### 1. Создать и активировать виртуальное окружение
Windows:
```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

macOS / Linux:
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. Установить зависимости
```bash
pip install -r requirements.txt
```

### 3. Применить миграции
```bash
python manage.py makemigrations
python manage.py migrate
```

### 4. Создать администратора
```bash
python manage.py createsuperuser
```

### 5. Заполнить демо-данными
```bash
python manage.py seed_demo
```

### 6. Запустить сервер
```bash
python manage.py runserver
```

Сайт:
- http://127.0.0.1:8000/

Админка:
- http://127.0.0.1:8000/admin/

## Как добавить ещё одного администратора

### Вариант 1: отдельный суперпользователь
```bash
python manage.py createsuperuser
```

### Вариант 2: создать обычного staff-пользователя через админку
1. Войти под суперпользователем.
2. Открыть раздел **Users**.
3. Создать пользователя.
4. Поставить галочку **Staff status**.
5. Выдать нужные права на модели:
   - categories
   - persons
   - text blocks
   - static pages
   - site settings

## Как менять тексты сайта через админку
- **Site Settings** — название сайта, SEO по умолчанию, футер, контакты.
- **Text Blocks** — маленькие редактируемые блоки:
  - `home_intro` — текст над карточками на главной
  - `home_bottom_text` — текст под карточками
  - `contacts_note` — доп. блок на странице контактов
- **Static Pages** — страницы `О сайте`, `Контакты`, `Политика конфиденциальности` и любые другие.
- **Categories** — описание категории и SEO категории.
- **Persons** — все поля карточки, включая обоснование, источники и редакционное примечание.

## Как добавить новую категорию
1. Админка → **Категории**
2. Нажать **Добавить**
3. Заполнить:
   - название
   - slug (можно оставить пустым)
   - краткое описание
   - полное описание
   - SEO
   - сортировку
4. Сохранить

## Как добавить новую карточку
1. Админка → **Карточки**
2. Нажать **Добавить**
3. Заполнить:
   - имя
   - категория
   - статус / метка
   - короткая подпись
   - изображение
   - описание
   - почему мы так считаем
   - дополнительная информация
   - источники / примечания
   - редакционное примечание
   - SEO
   - флаги публикации
4. Сохранить

## Логика контента
- Категории создаются только через БД / админку, ничего не зашито.
- Почти все тексты сайта можно менять без редактирования шаблонов.
- Если контент чувствительный, используйте поля:
  - категория
  - описание
  - почему мы так считаем
  - источники / примечания
  - редакционное примечание

## Безопасность для MVP
- Админка только по логину/паролю
- CSRF включён Django по умолчанию
- Шаблоны экранируют HTML по умолчанию
- Пароли пользователей хранятся в хешированном виде
- Для изображений есть ограничение по размеру и расширениям
- Для продакшена обязательно:
  - выключить `DEBUG`
  - сменить `SECRET_KEY`
  - указать `ALLOWED_HOSTS`
  - использовать HTTPS
